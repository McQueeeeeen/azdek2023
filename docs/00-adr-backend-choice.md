# ADR-0001: Выбор backend платформы

## Статус

Accepted

## Контекст

Нужен backend для modular monolith e-commerce с:

- быстрым MVP запуском
- чистой модульной архитектурой
- интеграциями оплаты и доставки
- дальнейшим ростом в B2C+B2B

## Решение

Выбран **NestJS + TypeScript**.

## Причины

- единый язык с frontend (TypeScript)
- сильная модульность и DI
- удобный API-first подход
- зрелые практики для webhook, очередей, RBAC, observability

## Последствия

- требуется дисциплина по слоистой архитектуре
- админка реализуется отдельным контуром (например, Next.js admin app)
